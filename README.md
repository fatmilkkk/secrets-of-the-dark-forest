# secrets-of-the-dark-forest

### Описание
SQL-исследование базы данных фэнтезийной игры (схема fantasy: users, race, events, items, 22 214 игроков, 1 307 678 транзакций). Цель — понять, как раса игрового персонажа влияет на покупку внутриигровой валюты «райские лепестки», и оценить покупательскую активность игроков.
* Рассчитана доля платящих игроков по всей базе и в разрезе расы персонажа (CTE + AVG на бинарном признаке payer)
* Построена статистика по чекам: сумма, мин/макс, среднее, медиана, стандартное отклонение (PERCENTILE_CONT, STDDEV)
* Найдена и измерена доля «нулевых» покупок через COUNT(*) FILTER (WHERE amount = 0)
* Определены самые популярные эпические предметы с долей продаж и долей уникальных покупателей (оконная функция SUM() OVER())
* Решена ad hoc-задача: зависимость активности игроков от расы персонажа — через цепочку из трёх CTE (total_players → paying_players → player_activity)

### Использованные приёмы
|Задача                        |Инструмент SQL                                    |
|------------------------------|--------------------------------------------------|
|Доля платящих игроков         |AVG() на бинарном поле, ROUND                     |
|Разрез по расе                |INNER JOIN + GROUP BY, CTE                        |
|Доля нулевых покупок          |COUNT(*) FILTER (WHERE …) вместо CASE             |
|Медиана чека                  |PERCENTILE_CONT(0.5) WITHIN GROUP                 |
|Доли продаж по предметам      |Оконная функция SUM(...) OVER ()                  |
|Активность по расам (ad hoc)  |Последовательность из 3 CTE + LEFT JOIN + COALESCE|

Пример решения одной из задач (полная версия — в Script-1.sql):
```sql
-- Доля платящих игроков по расам
WITH users AS (
    SELECT 
        r.race_id, r.race, 
        SUM(u.payer) AS race_paying_players,
        COUNT(*) AS race_total_players
    FROM fantasy.users u
    INNER JOIN fantasy.race r ON u.race_id = r.race_id 
    GROUP BY r.race_id, r.race  
)
SELECT *, ROUND((race_paying_players::numeric / race_total_players), 3) AS race_paying_share  
FROM users
ORDER BY race_paying_share DESC;
```

### Результаты

Из 22 214 зарегистрированных игроков платящими оказались 3 929 — общая конверсия 17,7%.

Разброс доли платящих между расами — от 17,1% (Elf) до 19,4% (Demon), всего 2,3 п.п. Это в пределах шума, поэтому раса персонажа не является значимым предиктором готовности платить.

Всего совершено 1 307 678 покупок. Стоимость покупок сильно смещена вправо: среднее — 525,69, медиана — всего 74,86, стандартное отклонение — 2517,35, максимум — 486 615,1 (почти в 6 500 раз выше медианы).

### Выводы и рекомендации
* Раса персонажа не является источником монетизации. Конверсия и активность покупок различаются между расами в пределах 2–3 п.п. — сегментировать маркетинг и акции по расе нецелесообразно, нужны другие признаки (поведенческие, временные, платёжные).
* Распределение стоимости покупок несимметрично — среднее завышено единичными крупными транзакциями. Для отчётности и принятия решений лучше опираться на медиану или межквартильный размах, а не на среднее значение чека.
* Стоит выделить «whale»-сегмент (игроков с аномально крупными разовыми покупками) в отдельную категорию и продумать для него отдельную VIP-механику монетизации, отличную от массового сегмента мелких частых покупок.
* Два предмета (6010 и 6011) формируют 97,7% продаж и куплены ~36% всех игроков. Такая концентрация — сигнал либо к проверке игрового баланса (не создают ли эти предметы слишком сильное преимущество, обесценивая остальной контент), либо к усилению маркетинга остальных предметов, чтобы расширить разнообразие покупок и продлить интерес к игре.
* Итог: основной доход обеспечивают немногочисленные крупные покупки и два «обязательных» предмета, а не различия между расами персонажей — фокус монетизации и удержания стоит сместить с сегментации по расе на управление выбросами в чеке и балансировку топовых предметов.
